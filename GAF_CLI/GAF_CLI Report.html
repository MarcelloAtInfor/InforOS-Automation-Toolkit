GAF_CLI — GenAI Agent Factory CLI

Python CLI toolkit for creating, managing, and invoking Infor OS GenAI tools and agents
PYTHON 3.x 6 CLI COMMANDS • 3 CORE MODULES MOST RECENT: BOM GENERATOR (16 TOOLS + 1 AGENT)
GAF_CLI is a standalone Python CLI that handles the full lifecycle of Infor OS GenAI assets — from local spec authoring and validation, through publishing to the GenAI platform, to invoking agents for testing. It provides a scriptable, repeatable alternative to manual UI-based asset management and was the sole deployment pipeline for the BOM Generator project.

CLI Commands

✓ validate_spec

Pre-flight check before publish. Enforces naming convention (GAF_*_Tool/Agent_v*), portability (no hardcoded URLs), type-specific field requirements, and content rules (no auth docs, example request required, debug template required).
python -m src.cli.validate_spec specs/tools/MyTool.json

🚀 publish_tool

Publishes a new tool spec to the GenAI platform. Wraps the spec in an array for POST /api/v1/tools. Returns the platform-assigned GUID on success.
python -m src.cli.publish_tool specs/tools/MyTool.json

🤖 publish_agent

Publishes a new agent spec (type TOOLKIT). Warns if type is not TOOLKIT. Agent's data.tools array references the tools it can orchestrate.
python -m src.cli.publish_agent specs/agents/MyAgent.json

🔧 update_asset

Full-replacement update via PUT /api/v1/tools/{guid}. Spec must contain the guid field from the original publish. Missing fields are removed (not merged).
python -m src.cli.update_asset specs/tools/MyTool.json

🔎 list_assets

Lists tools and agents on the platform. Filters by prefix or type. Separates tools (API_DOCS) from agents (TOOLKIT) in output. Returns name, GUID, and type.
python -m src.cli.list_assets --prefix GAF_SyteLine

▶ invoke_agent

Invokes any agent via the GenAI Chat API. Supports sync (chat_sync) and async (chat + polling) modes, interactive sessions, JSON output, and agent selection via -a flag.
python -m src.cli.invoke_agent -a MyAgent --async "prompt"

Features Developed During the BOM Generator Project
NEW

Async Chat Pattern

Eliminated the 240-second API gateway timeout. POST /chat returns immediately; GET /sessions/{id}/messages polls until the agent finishes. Supports long-running agents that take 15+ minutes across multiple chunks.

client.chat(prompt, tools, session)
client.poll_for_response(session_id, expected_count, poll_interval, max_polls)
NEW

Agent Selection Flag

Added -a / --agent argument to invoke_agent. Previously hardcoded to the Factory Agent. Now any deployed agent can be invoked without code changes. Default remains backward compatible.

python -m src.cli.invoke_agent \
  -a GAF_SyteLine_BomGenerator_Agent_v1 \
  "Create a wooden chair BOM"
ENHANCED

Type-Specific Validation

Validation now routes checks by asset type. Tools (API_DOCS) get api_docs content checks; Agents (TOOLKIT) get tools array and workflow checks. Type-name consistency enforced. Invalid types rejected early.

_Tool_v* → type: API_DOCS
_Agent_v* → type: TOOLKIT
Mismatch = validation failure
ENHANCED

Debug Template Enforcement

Every tool's responseInstructions must include a standardized debug block. On error, the GenAI model surfaces the exact URL called, payload sent, and response received — enabling rapid debugging without platform log access.

---DEBUG---
FULL_URL: [url]
FULL_PAYLOAD: [body]
FULL_RESPONSE: [response]
---END DEBUG---
INFRA

Session ID Discovery

Discovered that Infor's GenAI platform requires dashless hex session IDs. Dashed UUIDs silently fail (0 messages returned). uuid.uuid4().hex is now the standard. Pagination is 1-based (page 0 returns 422).

session = uuid.uuid4().hex
# "c3effe2f22a94caaa1595314719d00bd"
# NOT "c3effe2f-22a9-4caa-..."
INFRA

Response Format Normalization

GET /sessions/{id}/messages returns inconsistent formats (list vs paginated dict). get_session_messages() normalizes both into a consistent list, shielding callers from platform quirks.

msgs = client.get_session_messages(
  session_id, page=1, size=50
)
# Always returns list, newest-first

  • publish_agent.py — POST new agent
  • update_asset.py — PUT existing asset
  • list_assets.py — GET & filter
  • invoke_agent.py — sync/async chat

  • genai_client.py — GenAIClient class
  •   .create_tools() — POST array
  •   .update_tool() — PUT object
  •   .chat_sync() — blocking call
  •   .chat() — async fire-and-forget
  •   .poll_for_response() — polling loop
  •   .get_session_messages() — normalized

Shared — src/shared/

  • auth.py — OAuth2 password grant
  • config.py — .ionapi file loader
  • http_client.py — retries + backoff
  • validation.py — 12 validation rules
  • errors.py — static error factory
  • logging.py — credential redaction
