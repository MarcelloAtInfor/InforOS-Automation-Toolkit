DemoInvoiceLoader_V4 Delivery Report

Public handoff summary for the deterministic invoice automation rewrite.

Summary

DemoInvoiceLoader_V4 replaces the prior GenAI-driven transactional path with an RPA-driven, deterministic CSI API workflow. It was validated live for OCR, vendor reuse/create, item reuse/create, PO creation, PO-line creation, tax persistence, notifications, and fail-closed vendor ambiguity handling.

Highlights

  • Direct deterministic CSI IDO calls for vendor, item, PO, and PO-line logic.
  • Vendor scoring uses exact-name gating plus structured address matching with bounded typo tolerance.
  • Unsafe same-name cases fail closed instead of silently reusing or duplicating master data.
  • Tax writes to the visible SalesTaxT field.
  • Notifications include created asset details and unhandled failures still route to notification.

Public Setup Notes

  • Tracked source is intentionally scrubbed of tenant-specific values.
  • Use deploy.local.example.json and scripts/prepare_deploy.py to build a tenant-specific copy.
  • Rebind live OCR in Studio for the generated .deploy copy before production use.
